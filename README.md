## 1. 产品概述

### 1.1 产品背景

学海方舟的诞生源于一个真实而普遍的痛点——**计划写满本子，第三天就再也打不开了**。

从三年级开始，用户尝试过各种计划本、番茄钟App、四象限表格，每一种方法本身都很好，却始终无法坚持。问题的根源不在方法，而在工具：市面上的学习软件要么功能过于复杂，要么强制注册联网，要么完成后毫无反馈——做完了就做完了，没有成就感。

于是，一位不会写网页的准六年级学生，借助AI编程工具 WorkBuddy，用中文描述需求，让AI写出代码，自己担任“产品经理”，共同打造了这艘“永不靠岸的学习方舟”。

### 1.2 产品定位

学海方舟是一艘**为中小学生打造的、纯前端、零后端、全本地存储的学习计划工具**。它以“计划不白写”为初心，将三种经典学习方法论（GTD+番茄钟、四象限+费曼学习法、心流沉浸式学习）融入一套可视化、游戏化的交互体验中。

### 1.3 产品数据总览

| 维度 | 数据 |
|------|------|
| 代码行数 | 约 2,600 行（HTML 132 + CSS 882 + JS 1,500） |
| 文件数 | 3 个（index.html / style.css / app.js） |
| 外部依赖 | 零 —— 无 npm、无 CDN、无框架、无库 |
| 页面数 | 10 个 |
| 存储键 | 12 个 localStorage 键 |
| 成就徽章 | 16 枚 |
| 植物阶段 | 6 个成长阶段 |

### 1.4 目标用户

- **K12学生**：需要科学学习方法论但难以坚持的学习者
- **自驱型学习者**：希望量化学习成果、获得正向反馈的用户
- **隐私敏感用户**：不愿注册账号、不希望学习数据上传云端的人群


## 2. 设计哲学

### 2.1 核心设计原则

学海方舟的设计围绕四条核心原则展开：

**原则一：方法论驱动，而非功能堆砌**

每一个功能模块都对应一种经典的学习方法论，而非随意添加的功能。三大计划引擎分别对应 GTD+番茄钟、艾森豪威尔四象限+费曼学习法、心流理论+主题式学习。工具是载体，方法论才是内核。

**原则二：游戏化反馈，而非枯燥记录**

学习是反人性的，但游戏不是。XP经验值、电子盆栽成长、16枚成就徽章、四维技能树——每一次学习投入都能获得即时、可视化的正向反馈。让坚持变得“上瘾”。

**原则三：数据主权归于用户**

零注册、零联网、零上传。所有数据存储在用户自己的浏览器中，导出为 JSON 即可备份和迁移。学习数据是私密的，不需要任何人知道。

**原则四：极简技术，极致可靠**

零外部依赖、三个文件、纯前端部署。不需要服务器、不需要数据库、不需要网络。打开即用，永远在线。

### 2.2 产品故事隐喻

学海方舟将学习比作一场远航：

- **方舟**：承载学习旅程的容器，永不靠岸
- **三大引擎**：三种不同的航行方式——番茄流是划桨、四象限流是掌舵、沉浸流是扬帆
- **电子盆栽**：航行中积累的成长印记
- **成就徽章**：航行中获得的荣誉勋章

## 4. 三大计划引擎详解

### 4.1 知行·番茄流 ⏰（GTD + 番茄钟）

**核心机制**：GTD 宏观管控 × 番茄钟微观执行

#### 数据模型

```javascript
GTD 任务对象:
  id: Number        // 时间戳
  text: String      // 任务内容
  subject: String   // 学科：数学/语文/英语/科学/其他
  pomo: Number      // 预估番茄数
  done: Boolean     // 完成状态
  created: String   // 创建日期
```

#### 功能矩阵

| 功能 | 实现方式 | 状态 |
|------|----------|------|
| 收集箱输入 | input + 学科 select + 番茄数 input | ✅ |
| 任务列表渲染 | renderGTDTasks() 动态生成 | ✅ |
| 完成/删除 | 点击勾选切换 done 状态 | ✅ |
| GTD 四步展示 | 收集→整理→执行→回顾 | ✅ |
| 番茄钟大面板 | SVG 圆环进度 + 数字居中 | ✅ |
| Work/Break 自动切换 | 25min Work → 5min Break | ✅ |
| 番茄完成反馈 | 铃声 + Toast + XP+10 | ✅ |

#### 用户流程

```
输入任务 → 选择学科 → 设定番茄数 → 加入收集箱
                    ↓
    每日挑选高优先级 → 启动番茄钟（25+5）
                    ↓
        完成后勾选 ✓ → XP + 番茄计数 +1
                    ↓
        每周回顾 → 清理已完成 → 重新排期
```

### 4.2 明镜·四象限流 🪞（艾森豪威尔矩阵 + 费曼学习法）

**核心机制**：四象限筛选“学什么” × 费曼学习法确保“真学会”

#### 数据模型

```javascript
象限任务对象:
  text: String      // 任务内容
  subject: String   // 学科标签
  id: Number        // 时间戳

四个独立存储键: ark_q1 / ark_q2 / ark_q3 / ark_q4
```

#### 象限定义

| 象限 | 含义 | 颜色 | 操作提示 |
|------|------|------|----------|
| Q1 🔴 | 重要且紧急 | 红 #fca5a5 | 立即执行 |
| Q2 🟢 | 重要不紧急 | 绿 #86efac | 重点排期 |
| Q3 🟡 | 紧急不重要 | 黄 #fcd34d | 委派或快速处理 |
| Q4 ⚪ | 不重要不紧急 | 灰 #cbd5e1 | 果断剔除 |

#### 交互特性

- **拖放迁移**：使用 HTML5 Drag & Drop API，任务卡片可在四象限间自由拖拽
- **即时反馈**：拖放后数据即时保存，视图即时刷新，Toast 提示迁移目标
- **费曼引导**：四步引导卡片——选择概念→大白话教学→发现盲点→简化回顾
- **XP 联动**：添加四象限任务 +5 XP

### 4.3 墨滴·沉浸流 💧（心流触发 + 主题式学习）

**核心机制**：主题锚定 → 环境封印 → 心流冲刺 → 知识串联

#### 数据模型

```javascript
沉浸记录对象:
  theme: String        // 学习主题
  keywords: Array      // 关键词列表
  notes: String        // 学习笔记
  duration: Number     // 实际沉浸分钟数
  date: String         // 日期
```

#### 功能矩阵

| 功能 | 实现方式 | 状态 |
|------|----------|------|
| 主题锚定 | 文本输入 + 5个快速主题按钮 | ✅ |
| 时长选择 | 45/60/75/90 分钟四档 | ✅ |
| 心流计时器 | 紫色渐变，超时变绿 | ✅ |
| 四步指引 | 锚定→封印→冲刺→串联 | ✅ |
| 知识串联 | 关键词 + 笔记输入保存 | ✅ |
| 历史展示 | 最近6条倒序列表 | ✅ |

#### 心流计时器设计

深紫渐变背景（#0f0c29 → #302b63 → #24243e），4rem 大字号时间，紫色发光文字阴影，超过目标时长后数字自动变绿。

#### XP 联动

- 完成沉浸学习：+30 XP（最大单次 XP 来源）
- 保存沉浸记录：+10 XP


## 5. 学习工具模块

### 5.1 错题本 📕（SRS 间隔复习）

**算法**：简化版 SM-2 间隔重复

#### 复习机制

用户自评三个档位：
- ❌ “忘了”（q=0）→ interval = 1（明天复习）
- 🌊 “模糊”（q=2）→ interval = 1（重置）
- ✅ “记住”（q=5）→ EF 调整 + interval 更新

#### 数据模型

```javascript
错题卡片对象:
  question: String
  answer: String
  note: String        // 错误原因
  subject: String     // math/chinese/english/science
  interval: Number    // 当前间隔天数
  efactor: Number     // 简易系数，初始 2.5
  nextReview: String  // 下次复习日期
  created: String
```

#### 学科配色

| 学科 | CSS类 | 色条 |
|------|-------|------|
| 数学 | math | 蓝 #3b82f6 |
| 语文 | chinese | 粉 #ec4899 |
| 英语 | english | 绿 #10b981 |
| 科学 | science | 黄 #f59e0b |

#### XP 联动

- 添加错题：+8 XP
- 复习“记住”：+12 XP

### 5.2 每日复盘 📝

#### 数据模型

```javascript
复盘记录对象:
  date: String        // 日期（同一天覆盖）
  mood: String        // 5档 Emoji
  goals: String       // 目标完成情况
  insight: String     // 学习感悟
  feynman: String     // 费曼输出
  plan: String        // 明日计划
```

#### 设计亮点

- **费曼输出框**：专属视觉样式（橙色虚线边框 + 浅橙底色），引导用户输出式学习
- **心情选择**：5 档 Emoji（😄😊😐😔😤）
- **覆盖保存**：同一天可多次保存，覆盖旧记录

#### XP 联动

保存复盘：+20 XP

### 5.3 知识图谱 🗺

**数据来源**：墨滴·沉浸流中保存的所有 `immerse_records`

#### 关键词云

- 从所有沉浸记录中提取关键词，按出现频率排序
- 7 色调色板（蓝紫粉绿黄红青）
- 字号随频率动态增大（0.75 + count × 0.08 rem）
- 带边框 + 悬浮缩放效果

#### 主题脉络

- 倒序时间线显示所有沉浸记录
- 每条记录显示日期、时长、主题、笔记、关键词标签

## 7. 技术实现

### 7.1 运行机制

```
用户打开 index.html
    │
    ▼

```

### 7.2 动画系统

| 动画 | 技术 | 描述 |
|------|------|------|
| 封面入场 | CSS coverFadeIn | 1.2s 淡入 + 上浮 30px |
| 帆船浮动 | CSS shipFloat | 3s 周期上下浮动 12px |

### 7.3 白噪音技术（纯 Web Audio API 合成）

完全通过 Web Audio API 程序合成，零外部音频文件依赖：

| 类型 | 噪声算法 | 滤波器 | 特效 |
|------|----------|--------|------|
| 🌧 雨声 | 白噪声 | Bandpass 2400Hz Q0.5 | 淡入 0.5s |
| 🌊 海浪 | 棕色噪声（一阶低通反馈） | Lowpass 600Hz | LFO 0.12Hz 调制增益 |
| 💨 风声 | 粉红噪声（Voss-McCartney 7阶） | Lowpass 900Hz | 淡入 0.5s |

**音频节点链**：
- 雨声：source → BiquadFilter(bandpass) → Gain → destination
- 海浪：source → BiquadFilter(lowpass) → Gain（LFO调制）→ destination
- 风声：source → BiquadFilter(lowpass) → Gain → destination

4秒循环 buffer，无缝播放；切换时自动停止上一个；AudioContext suspend 自动 resume。

- 12个键全量导出为 JSON 文件

## 11. 结语

学海方舟 v1.0 以 **2,600 行零依赖代码** 实现了 10 个页面、3 个计划引擎、5 个学习工具模块、完整的游戏化成长系统（XP/盆栽/技能/成就/统计）。

其核心亮点在于：

1. **知行合一**：GTD 宏观管控 + 番茄钟微观执行
2. **优先级精学**：四象限筛选 + 费曼学习法确保真学会
3. **深度浸泡**：心流触发 + 主题式沉浸学习
4. **数据主权**：全本地存储，零注册零联网

当前版本的短板主要集中在三处：**部分内存状态不持久化、技能树不完整、知识图谱缺少真正可视化**。这些均可在不改动架构的前提下增量修复。

> ⛵ **方舟永远在建造。每一次学习，都是一次远航。**
